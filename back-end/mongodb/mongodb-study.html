# mongodb入门

# 1.安装mongodb

本文除了1.2节的环境为centos外，其余操作均在Mac中进行

# 1.1 Mac环境

打开终端，使用homebrew直接安装

$ brew install mongodb

安装完成后，创建数据库存储目录，mac默认以下路径

$ sudo mkdir -p /data/db

给予该目录权限，不然用权限方式启动时会报错

$ sudo chown -R 你的用户名 /data/db

# 1.2 centos环境

下载安装包

curl -O https://fastdl.mongodb.org/linux/mongodb-linux-x86_64-3.2.12.tgz

解压

tar -zxvf mongodb-linux-x86_64-3.2.12.tgz

移动到指定位置

mv  mongodb-linux-x86_64-3.2.12/ /usr/local/mongodb

在/usr/local/mongodb下创建文件夹

mkdir -p /data/db
mkdir  /logs

在/usr/local/mongodb/bin下新建配置

vi mongodb.conf

dbpath=/usr/local/mongodb/data/db #数据文件存放目录
logpath=/usr/local/mongodb/logs/mongodb.log #日志文件存放目录
port=27017 #端口
fork=true #以守护程序的方式启用，即在后台运行
nohttpinterface=true
auth=true
bind_ip=0.0.0.0

环境变量配置

vi /etc/profile

export MONGODB_HOME=/usr/local/mongodb
export PATH=$PATH:$MONGODB_HOME/bin

保存后，重启系统配置

source /etc/profile

# 2.初次启动mongodb

启动mongodb

$ mongod

值得注意的是，如果没有为mongodb创建环境变量，则需进入mongodb的bin文件夹下，执行./mongod

$ cd /usr/local/Cellar/mongodb/3.6.3/bin
$ ./mongod

第一次启动后会报一些警告，例如

018-04-02T19:53:37.674+0800 I CONTROL  [initandlisten] ** WARNING: Access control is not enabled for the database.

这是因为新版本的MongDB增加了安全性设计，推荐用户创建使用数据库时进行验证

如果用户想建立简单连接，则会提示警告信息

# 3.添加管理用户

首先进入交互界面

$ mongo
MongoDB shell version v3.6.3
connecting to: mongodb://127.0.0.1:27017
MongoDB server version: 3.6.3

切换到admin数据库

> use admin

设置管理用户

> db.createUser({user:"admin",pwd:"123456",roles:[{role: "userAdminAnyDatabase", db:"admin"}]})

# 4.使用权限方式启动mongodb

添加完管理用户后，可以使用db.shutdownServer()关闭MongoDB

这里注意不要使用kill直接去杀掉mongodb进程，否则请去data/db目录下删除mongo.lock文件

然后用权限方式启动mongodb，注意日志路径需要指定到一个文件，而不是文件夹

输入命令后提示以下信息则表示正常

$ mongod --dbpath=/data/db --logpath=/data/logs/db.log --fork --auth
about to fork child process, waiting until server is ready for connections.
forked process: 10005
child process started successfully, parent exiting

再次使用mongo进入交互界面，这时候不会有警告了

$ mongo
MongoDB shell version v3.6.3
connecting to: mongodb://127.0.0.1:27017
MongoDB server version: 3.6.3

仍然切换到admin数据库，并用刚才创建的账户来验证，成功的话会显示1

> use admin
switched to db admin
> db.auth("admin","123456")
1

# 5.创建操作用户

验证之后还是做不了操作，因为admin只有用户管理权限，还需创建操作用户

新建一个数据库nodedb，创建操作用户node

> use nodedb
switched to db nodedb
> db
nodedb
> db.createUser({user:"node",pwd:"123456",roles:[{role:"readWrite",db:"nodedb"}]})
Successfully added user: {
	"user" : "node",
	"roles" : [
		{
			"role" : "readWrite",
			"db" : "nodedb"
		}
	]
}

然后进行验证，成功同样返回1

> db.auth("node","123456")
1

# 6.mongoose连接数据库

这里采用express框架进行mongoose的操作，目录结构为：

project
  |--bin
  |--models
     |--mongo.js
     |--user.js
  |--public
     |--images
     |--javascripts
     |--stylesheets
  |--routes
     |--api.js
     |--index.js
  |--views
     |--index.html
     |--error.html
     |--api.html
  |--app.js
  |--config.js
  |--package.json
  |--package-lock.json

首先安装mongoose

$ npm install mongoose --save

在项目根目录下新建一个/models/mongo.js

var mongoose = require("mongoose");
var db = mongoose.connect('mongodb://node:p123456@127.0.0.1:27017/nodedb');
module.exports = db;

在app.js引入

var db = require('./models/mongo');

# 7.创建数据模型

在项目根目录下新建/models/user.js

var mongoose = require("mongoose");
//一个用户模型
var userSchema = new mongoose.Schema({
	username    : {type: String},
	password    : {type: String},
	isAdmin     : {type: Boolen},
	registerTime: {type: String}
});
//创建Model
var userModel = mongoose.model('User', userSchema );
module.exports = userModel;

这里先提一下，在这部分代码被引入和运行后，进入交互界面的nodedb数据库中，显示所有集合

> db
nodedb
> show collections
users

发现新增了一个名为users的集合，这是在创建数据模型时自动生成的

集合的命名遵从英文的复数规则，比如mongoose.model('User', userSchema )对应的集合名为users，而mongoose.model('Fish', userSchema )对应的集合名为fish

# 8.增删改查

这里简单提一下mongoose的增删改查，实际使用中会复杂很多，比如mongoose不建议直接使用update来更新数据

在routes中新建api.js

var user = require('../models/user');
var express = require('express');
var router = express.Router();
var sd = require('silly-datetime');//日期&时间模块，需npm install

router.get('/', function(req, res) {
	res.render('api');
});

在app.js中引入，并将/api的路由交由该模块处理

var apiRouter = require('./routes/api');
...
app.use('/api', apiRouter);

# 8.1 新增用户

回到api.js

router.post('/addUser', function (req, res) {
	//查找用户是否已存在
	user.findOne({username: req.query.username}, function (err, user) {
		if (err) {
			throw err;
		}
		if (user) {
			res.json({
				success: 0,
				message: 'User has already existed.'
			});
		}
	});
	//新建用户
	let newUser = new user();
	newUser.username = req.body.username;
	newUser.password = req.body.password;
	newUser.isAdmin = false;
	newUser.registerTime = sd.format(new Date(), 'YYYY-MM-DD HH:mm');
	newUser.save(function(err) {
		if (err) {
			res.send(err);
		}
		res.json({
			success: 1,
			message: 'User' + req.body.name + 'is created!'
		});
	});
});

# 8.4 查找用户

获取所有用户

router.get('/getUsers', function(req, res) {
	user.find(function(err, users) {
		if (err) {
			res.send(err);
		}
		res.json({
			success: 1,
			message: '',
			data: users
		});
	});
});

查找单个用户

router.get('/findUser', function(req, res) {
	user.findOne({name: req.query.name}, function (err, user) {
		if (err) {
			throw err;
		}
		if (!user) {
			res.json({
				success: 0,
				message: 'User is not found.'
			});
		}
		else if (user) {
			res.json({
				success: 1,
				message: '',
				data: user
			});
		}
	});
});
